# eCTF 2026 — Security Requirements

Security requirements are NOT tested during Handoff. Other teams earn points by exploiting failures to meet these requirements by capturing Attack Phase Flags.

> **This is an *embedded* security competition.** When building your design, there are subtle attack vectors that apply specifically to embedded designs.

## Security Requirement 1: Permission Enforcement
**An attacker should not be able to perform any file action without a validly provisioned HSM with the permissions to perform that action on files belonging to that group.**

- Attacker should not be able to **read** files without read permission
- Attacker should not be able to **create** files without write permission
- Attacker should not be able to **receive** files without receive permission
- Receive permission also applies to **interrogate**: the interrogated device should only return metadata about files for which the requesting device has the receive permission

## Security Requirement 2: PIN Protection
**No PIN-protected action should be able to be completed by a user without prior knowledge of the PIN.**

This includes **confidentiality** of the PIN. The HSM should not expose information about its PIN to any unauthorized user.

## Security Requirement 3: File Integrity
**An HSM should not successfully receive any file that was not generated by another valid HSM with write permissions for that group.**

This includes protecting file integrity from being compromised in any way by an attacker.
